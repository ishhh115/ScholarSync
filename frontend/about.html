About ScholarSync

An AI-powered research discovery platform built with Sentence Transformers, FAISS vector search, and XGBoost classification.

Project Journey
1
476 papers collected
Fetched real research papers from arXiv API across 5 topics: ML, NLP, CV, RL, Deep Learning
2
384-dimensional embeddings generated
Converted each paper abstract into a dense vector using all-MiniLM-L6-v2 Sentence Transformer model
3
FAISS index built
Indexed all 476 embeddings in a FAISS IndexFlatIP for fast cosine similarity search at query time
4
3 ML models trained
Trained Logistic Regression, Random Forest, and XGBoost on embeddings to classify papers by topic
5
XGBoost achieved 0.70 F1
XGBoost outperformed other models with weighted F1 of 0.7044. Strong on cs.LG, cs.CV, cs.CL
6
Semantic search deployed
FastAPI backend serving search, classification, and recommendation endpoints. Frontend built with vanilla JS

Why XGBoost Won
Non-linear Boundaries
Logistic Regression assumes a linear decision boundary. Paper embeddings in 384D space have complex non-linear structure — XGBoost captures this with decision trees.
Feature Interactions
XGBoost automatically learns interactions between embedding dimensions. A paper about "vision transformers" activates both CV and NLP features — XGBoost handles this, LR can't.
Boosting
XGBoost builds trees sequentially, each one correcting the previous model's mistakes. This iterative error correction is why it outperforms a single Random Forest on this dataset.
💡 Bottom line: Embeddings create a high-dimensional space with complex structure. Tree-based boosting navigates this better than linear models — which is why XGBoost consistently outperforms on tabular/embedding tasks.
